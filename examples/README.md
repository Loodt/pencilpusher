# pencilpusher Example Demo

Demonstrates the full pipeline using a fictional user: **Jane Thandeka Moyo**, a Zambian-born, SA-resident mining engineer.

## What's included

```
examples/
├── sources/                          ← Fictional source documents
│   ├── jane_moyo_id.md               ← Zambian NRC (ID card)
│   ├── jane_moyo_passport.md         ← SA passport
│   └── moyo_mining_pacra.md          ← PACRA company printout
├── inbox/                            ← Forms to fill
│   ├── application_form.docx         ← DOCX table-based form
│   └── application_form.pdf          ← PDF flat form (same fields)
├── expected/                         ← Filled output (generated by demo)
│   ├── application_form_filled.docx
│   └── application_form_filled.pdf
├── run_demo.py                       ← Run this!
└── README.md                         ← You are here
```

## Run the demo

```bash
# No API key needed — demo uses pre-extracted data
python examples/run_demo.py
```

Then open both files in `expected/` to verify:
- DOCX: data filled into table cells, original styling preserved
- PDF: data inserted at label positions, original layout preserved

## What this proves

1. **DOCX table filling** — python-docx writes into empty cells adjacent to labels
2. **PDF text insertion** — PyMuPDF inserts text at detected positions (no form fields needed)
3. **Same data, two formats** — identical information fills both document types
4. **Style preservation** — original fonts, sizes, borders, headers all survive

## The full pipeline (with API key)

With `ANTHROPIC_API_KEY` set, you can test the real LLM-powered pipeline:

```bash
pencilpusher init
# Copy sources to ~/.pencilpusher/sources/
pencilpusher ingest-all     # Claude extracts data → builds wiki
pencilpusher show identity  # Check what was extracted
# Copy forms to ~/.pencilpusher/inbox/
pencilpusher fill-all       # Claude matches fields → fills forms
```
